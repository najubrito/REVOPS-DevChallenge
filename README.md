## Dev Challenge – REVOPS
## Contexto
Este projeto faz parte de um desafio para a vaga de RevOpss.

## O que fiz:
Mesmo não finalizando todas as tasks, os seguintes pontos foram efetivamente trabalhados:

Configurei o HttpClient utilizando Dependency Injection (AddHttpClient)
Removi a causa do erro:
“The method or operation is not implemented.”
Centralizei a BaseAddress via AppSettings
Entendi o fluxo de autenticação da API
Configurei e utilizei meu token de desenvolvimento
Analisei a documentação do Talk (Swagger)

## O que não fiz:

As tasks abaixo não foram totalmente implementadas, principalmente por limitação de tempo e priorização do entendimento da base:
Refatoração completa da lógica do Home.razor para uma camada de serviço reutilizável
Criação de componentes reutilizáveis para a UI
Evolução visual significativa da interface (UI)
Implementação do histórico de chats pesquisados
Correção estrutural da tabela para permitir múltiplas consultas do mesmo chat
Exibição completa de dados do contato (email, tags, datas, etc.) na UI
Apesar disso, essas tasks foram analisadas conceitualmente, e tenho clareza de como as implementaria em um próximo ciclo.

## Considerações 

Mesmo sabendo que, no momento da entrega do código, eu não atendo aos critérios da vaga, encarei este desafio como uma oportunidade de aprendizado.
